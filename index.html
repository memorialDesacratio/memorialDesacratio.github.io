🕵️ OSINT

Что такое OSINT?

OSINT (Open Source Intelligence) — разведка по открытым источникам. Это сбор и анализ информации, которая доступна любому желающему: сайты, соцсети, форумы, утекшие базы, публичные документы.

Ключевые принципы

  • Легальность — используем только открытые данные
  • Верификация — перекрёстная проверка из нескольких источников
  • Автоматизация — скрипты и тулы для ускорения сбора
  • Анализ — связывание точек: данные → информация → выводы

Основные инструменты

  • Maltego — визуализация связей между данными (графы)
  • Shodan — поиск устройств, подключённых к интернету
  • theHarvester — сбор email, поддоменов, IP-адресов
  • Recon-ng — мощный фреймворк для веб-разведки
  • SpiderFoot — автоматический OSINT-сканер с 200+ модулями
  • Google Dorks — операторы для точного поиска

Google Dorks (примеры)

# Поиск PDF-документов на сайте
site:example.com filetype:pdf

# Поиск уязвимых страниц
inurl:admin.php

# Поиск по заголовку
intitle:"index of" /backup

# Поиск email-адресов
"@example.com" filetype:xls

Техники поиска людей

  • Поиск по username — namechk, whatsmy name
  • Поиск по соцсетям — Facebook, VK, Telegram, LinkedIn
  • Поиск по форумам — Google "username site:forum.ru"
  • Поиск по утечкам — haveibeenpwned, leakcheck

Этика OSINT

OSINT — это сбор открытых данных. Не путай с взломом. Не используй данные для вреда. С разрешения или в исследовательских целях.

⚔️ Пентестинг

Что такое пентест?

Penetration Testing — легальная имитация атаки на систему с целью поиска уязвимостей. Главное отличие от реальной атаки — есть договор и рамки.

Этапы пентеста (PTES)

  1. Reconnaissance — сбор информации о цели
  2. Scanning & Enumeration — сканирование портов, сервисов, версий
  3. Exploitation — эксплуатация найденных уязвимостей
  4. Privilege Escalation — повышение привилегий
  5. Post-Exploitation — закрепление, сбор данных
  6. Reporting — детальный отчёт с рекомендациями

Инструменты

  • Nmap — сканер портов. Базовый навык
  • Metasploit — фреймворк с готовыми эксплойтами
  • Burp Suite — перехват и анализ HTTP/HTTPS трафика
  • Hydra — брутфорс паролей (FTP, SSH, HTTP)
  • Wireshark — анализ сетевого трафика
  • John the Ripper / Hashcat — взлом хэшей
  • Gobuster / Dirb — перебор директорий на веб-сервере

Практика (легальные полигоны)

  • HackTheBox — реалистичные машины для пентеста
  • TryHackMe — обучение с нуля
  • VulnHub — уязвимые виртуальные машины
  • PentesterLab — веб-пентест

Методологии

  • OWASP Top 10 — веб-уязвимости
  • OSSTMM — методология тестирования
  • NIST SP 800-115 — руководство по пентесту

💻 Программирование

Мой стек

  • Python — боты, парсеры, скрипты, OSINT-тулы, автоматизация
  • JavaScript (Vanilla) — фронтенд, анимации, интерактив
  • Node.js / Express — бэкенд, API, веб-сервера

  • SQL / SQLite — базы данных, запросы, агрегации
  • HTML + CSS — адаптивные сайты, анимации, темы

Полезные концепции

  • DRY — Don't Repeat Yourself. Выноси повторяющееся в функции
  • KISS — Keep It Simple, Stupid. Не усложняй
  • SOLID — принципы ООП для больших проектов
  • Git — каждый проект в репозитории. Коммиты — часто

Best practices

  • Всегда используй виртуальное окружение (venv)
  • Не храни секреты в коде — используй .env
  • Пиши README — потом скажешь себе спасибо
  • Комментарии — это норма. Но лучше читаемый код без них

Полезные ссылки

  • Learn X in Y Minutes — быстрые туториалы
  • Roadmap.sh — дорожные карты разработчика
  • Codewars / LeetCode — задачи для практики

🐧 Linux / Unix

Мои дистрибутивы

  • Arch Linux — основной дистр. Полный контроль, rolling release.
  • Kali Linux — для пентеста и OSINT. Всё предустановлено.
  • Debian — для серверов. Стабильность и надёжность.

Базовые команды (must know)

# Навигация
ls -la          # список файлов
cd /path        # переход
pwd            # текущая директория

# Файлы
cp -r src dst   # копирование
mv src dst      # перемещение/переименование
rm -rf dir      # удаление
chmod +x file   # сделать исполняемым

# Процессы
ps aux          # все процессы
htop/top        # мониторинг
kill -9 PID     # убить процесс

# Сеть
ip a            # сетевые интерфейсы
ss -tulpn       # открытые порты
ping host       # проверка связи

Docker

Контейнеризация — стандарт. Каждый сервис в своём контейнере. Docker Compose для связки.

# Основные команды
docker ps                     # запущенные контейнеры
docker images                 # образы
docker run -d --name web nginx
docker compose up -d          # запустить проект

Полезные файлы

  • /etc/passwd — пользователи
  • /etc/shadow — хэши паролей
  • /var/log/ — системные логи
  • /etc/nginx/ — конфиги nginx
  • ~/.bashrc / ~/.zshrc — конфиг шелла

🪟 Windows

Windows в контексте безопасности

Windows — самая популярная ОС, а значит — самая атакуемая. Понимание её внутренностей критично.

PowerShell

Современная командная оболочка Windows. Обязателен к изучению.

# Информация о системе
Get-ComputerInfo

# Сеть
Get-NetTCPConnection
Test-NetConnection google.com

# Процессы
Get-Process
Stop-Process -Name notepad

# Логи безопасности
Get-WinEvent -LogName Security -MaxEvents 50

Active Directory

  • Централизованное управление пользователями и компьютерами
  • Group Policy — массовая настройка системы
  • Kerberos — протокол аутентификации

Безопасность Windows

  • Windows Defender — базовая защита
  • BitLocker — шифрование диска
  • AppLocker — контроль запуска приложений
  • Windows Firewall — сетевой экран

🛡️ Кибербезопасность

База безопасности

  • 2FA — двухфакторка везде, где можно
  • Менеджер паролей — Bitwarden, KeePass
  • Обновления — всегда вовремя
  • Принцип最小 — минимум прав, минимум риска

Шифрование

  • GPG — шифрование email и файлов
  • LUKS — шифрование диска в Linux
  • SSL/TLS — защита канала связи
  • HTTPS Everywhere — всегда HTTPS

Защита данных

  • Регулярные бэкапы (3-2-1 правило)
  • Шифрование чувствительных файлов
  • Не храни пароли в браузере
  • Проверяй утечки на haveibeenpwned.com

Threat Intelligence

Следи за угрозами: MITRE ATT&CK, CVE reports, threat feeds.

🌐 Сети / Networking

Основы TCP/IP

  • IP — адресация устройств в сети
  • TCP — надёжная доставка с подтверждением
  • UDP — быстрая доставка без гарантии
  • DNS — преобразование доменов в IP
  • HTTP/HTTPS — веб-трафик

Порты (основные)

21   — FTP
22   — SSH
25   — SMTP
53   — DNS
80   — HTTP
443  — HTTPS
3306 — MySQL
8080 — HTTP-альтернатива

Прокси и VPN

  • HTTP прокси — только веб-трафик
  • SOCKS5 — любой трафик
  • OpenVPN — шифрованный туннель
  • WireGuard — современный и быстрый VPN
  • Tor — анонимизация через цепочку нод

Анализ трафика

Wireshark — де-факто стандарт. tcpdump для CLI. Анализируй, фильтруй, понимай.

🔐 Шифрование / Crypto

Основы криптографии

  • Хэширование — MD5 (устарел), SHA-256, SHA-3
  • Симметричное — AES, ChaCha20 (один ключ)
  • Асимметричное — RSA, ECC (публичный + приватный ключи)
  • Подпись — подтверждение авторства

GPG (GNU Privacy Guard)

# Создать ключ
gpg --full-generate-key

# Зашифровать файл
gpg -e -r "Имя" file.txt

# Расшифровать
gpg -d file.txt.gpg

# Подписать
gpg -s file.txt

SSL/TLS

Протоколы шифрования канала связи. HTTPS = HTTP + TLS. LetsEncrypt — бесплатные сертификаты.

Практические советы

  • Хэшируй пароли (bcrypt, argon2)
  • Не придумывай свою криптографию
  • Используй проверенные библиотеки

🎭 Анонимность

Принципы анонимности

  • OPSEC — операционная безопасность. Не свети лишнего
  • Разделение — отдельные профили для разных активностей
  • Минимум данных — не оставляй цифровой след без необходимости

Инструменты

  • Tor Browser — анонимный веб-сёрфинг
  • VPN — шифрование трафика, смена IP
  • Tails OS — анонимная ОС с RAM-only режимом
  • ProtonMail / Tutanota — приватная почта

Метаданные

Фотографии содержат EXIF-данные: GPS, камера, дата. Удаляй перед публикацией.

# Удалить метаданные (Linux)
exiftool -all= photo.jpg

# Или через ImageMagick
convert photo.jpg -strip photo_clean.jpg

Чего не делать

  • Не использовать один пароль везде
  • Не входить в личные аккаунты через Tor
  • Не открывать документы из подозрительных источников
  • Не верить в полную анонимность — её не существует
